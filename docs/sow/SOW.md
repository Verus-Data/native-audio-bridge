## 1. Project Description

A native macOS voice interaction layer that enables always-on hot word detection, speech-to-text, and text-to-speech capabilities with private, in-memory processing. Commands are dispatched via webhook for action execution or logged for conversation, ensuring privacy and low latency.

---

## 2. Objectives

### 2.1 Core Goals
- Implement always-on audio listening without key presses
- Process audio entirely in memory (no disk persistence)
- Dispatch commands via OpenClaw webhook format
- Support optional conversation logging via user-configurable output
- Establish baseline silence detection at 1.5s with architectural support for future moving average

### 2.2 Success Metrics
- Hot word detection within 500ms of utterance
- Command processing latency < 1s after speaking
- Webhook delivery success rate > 99%
- Zero audio persistence to disk
- Memory footprint < 80MB during active listening

---

## 3. Scope

### 3.1 In Scope
- Always-on microphone monitoring using AVAudioEngine
- Hot word detection ("hey claW") via SFSpeechRecognizer
- Command capture with 1.5s baseline silence detection
- Webhook dispatch to OpenClaw using standard format
- User-configurable output destination (webhook or JSONL file)
- Optimized CPU/memory footprint for background operation

### 3.2 Out of Scope
- Moving average pause detection (Phase 2)
- Advanced pause reasoning during speech
- Multiple hot word support
- Extensive UI components
- Cross-platform support (iOS/macOS only initially)

## 5. Functional Specification

### 5.1 State Machine
```
┌─────────┐     Hot Word      ┌──────────┐     Silence      ┌─────────┐
│  IDLE   │ ─────────────────▶│ LISTENING│ ────────────────▶│ PROCESS │
│ (Always │    Detected       │(Capture  │    Detected      │ (STT)   │
│   On)   │                   │ Command) │                  │         │
└─────────┘                   └──────────┘                  └────┬────┘
     ▲                                                          │
     │                                                          ▼
     │                                                    ┌─────────┐
     │                                                    │ DISPATCH│
     │                                                    │(Webhook │
     │                                                    │ or Log) │
     │                                                    └────┬────┘
     │                                                         │
     └─────────────────────────────────────────────────────────┘
                            TTS Response (if any)
```

### 5.2 Data Flow
Audio Capture → Hot Word Detection → Command Capture → STT → [Webhook Dispatch] / [JSONL Output] → TTS

### 5.3 Configuration
- **Settings File**: `~/.native-audio-bridge/config.yaml`
  ```yaml
  hot_word: "hey claW"
  silence_timeout: 1500
  output_destination: "webhook"
  webhook_url: "https://gateway.openclaw.io/hooks/agent"
  webhook_token: "{{autogenerated_token}}"
  log_file: "~/audio-bridge.log"  # when destination=file
  ```

### 5.4 Error Handling
- Retry webhook delivery (exponential backoff, max 3) on network failure
- Local state preservation across reboots (in-memory reset is acceptable)
- User notification on critical failures via OpenClaw webhook
